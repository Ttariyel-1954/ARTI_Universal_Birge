1 Nə üçün Universal AI Agent?

Azərbaycanda və dünyada kiçik və orta biznes (SMB) sektoru iqtisadiyyatın onurğa sütunudur. Lakin bu bizneslər çox vaxt modern AI texnologiyalarından kənarda qalırlar — çünki fərdi bir həll yaratmaq bahalı, hazır həllər isə həmişə onların spesifik ehtiyaclarına uyğun olmur.

1.1. Problem: eyni ssenari, fərqli sahələr

Baxın, bir tikinti şirkəti, bir aptek şəbəkəsi və bir logistika firması — zahirən fərqli işlər görürlər. Lakin onların informasiya sistemlərinin özəyinə diqqət yetirsək, heyrətamiz bir oxşarlıq görərik:

Fəaliyyət Tikinti İdarəsi Aptek Şəbəkəsi Logistika
Obyektlərin idarə olunması Obyektlər, inşaat sahələri Filiallar, anbarlar Depolar, marşrutlar
İnventar / resurs Materiallar, avadanlıq Dərmanlar, ehtiyat Mal-yük, vasitələr
İnsan resursları Briqadalar, fəhlələr Əczaçılar, konsultantlar Sürücülər, ekspeditorlar
Maliyyə axını Smeta, hesablaşma Satış, qaimələr Fraxt, komisyon
AI istifadə halı Smeta təhlili, risk proqnozu Sifariş proqnozu, müşteri sorğuları Marşrut optimizasiyası

Bax bu oxşarlıq bizə bir möhtəşəm fürsət verir: bir dəfə yaxşı qurulmuş skelet → onlarla biznes sahəsinə uyğunlaşdırıla bilər. Bu fəlsəfəyə "Universal Agent" deyirik.

i
Bu nə üçün həm də Anthropic Claude əsrinə uyğundur?

Müasir LLM-lər (Claude, GPT-4, vs.) bir ümumi zəka təmin edir. Biznesə tətbiq etmək üçün bizə kontekst lazımdır — məlumat bazası, biznesin qaydaları, istifadəçi interfeysi. Bizim universal agent modeli məhz bu "kontekst qabığı"nı təmin edir. LLM-ə yalnız domen məlumatını dəyişməklə biz onu bir sahədən digərinə keçirə bilərik.

1.2. Nə üçün Təmir-Tikinti modeldir?

TTİS (Təmir-Tikinti İdarəsi Sistemi) xoşbəxtlikdən ən zəngin domen sahəsidir: orada layihələr, smetalar, müqavilələr, briqadalar, materiallar, avadanlıq, tərəddadlar, monitorinq, hesabat — hamısı bir yerdədir. Bunu qurduqdan sonra hər hansı sadə domenə keçid asanlaşır.

4 Domen-dəyişən prinsipi

Bu layihənin ən əhəmiyyətli cəhəti — universallıq. Bunu necə texniki olaraq təmin edirik? "Domen modulları" (domain modules) yanaşması ilə.

4.1. Domen nədir?

Domen — sizin biznesin həll etdiyi konkret problem sahəsidir. Məsələn:

  • Tikinti domeni — layihələr, smetalar, briqadalar, materiallar
  • Aptek domeni — filiallar, dərmanlar, resept, satış
  • Təhsil domeni — məktəblər, kurrikulum, şagirdlər, imtahanlar

4.2. Strategiya: "Core + Plugin"

Biz layihəni iki hissəyə bölürük:

Hissə Nə edir? Dəyişirmi?
Core (Özək) Autentifikasiya, baza bağlantısı, loglama, API gateway, istifadəçi idarəsi, rol-əsaslı icazə ❌ Heç vaxt dəyişmir
Domain (Domen) Konkret iş sahəsinin cədvəlləri, formaları, hesabatları, biznes qaydaları ✅ Sahəyə görə dəyişir
Shared (Ortaq) Ümumi komponentlər: fayl yükləmə, bildiriş, PDF eksport, cədvəl komponenti ⚠️ Az-az dəyişir

Backend-də bu prinsip belə reallaşır (bunu Dərs 3-də praktiki görəcəyik):

backend/src/server.js — domen yükləmə prinsipi 
// ═══ CORE qatı — heç vaxt dəyişmir ═══
const express = require('express');
const app = express();

require('./core/middleware/auth')(app);     // autentifikasiya
require('./core/middleware/logging')(app);  // loglama
require('./core/routes/users')(app);        // istifadəçi idarəsi

// ═══ DOMAIN qatı — sahəyə görə dəyişir ═══
const domain = process.env.DOMAIN || 'construction';
require(`./domain/${domain}/routes`)(app);

// ═══ SHARED qatı — ümumi xidmətlər ═══
require('./shared/services/files')(app);
require('./shared/services/ai')(app);       // Claude API

app.listen(3001);
✓
Nəticə

Aptek şəbəkəsinə keçid üçün yalnız DOMAIN=pharmacy environ dəyişəni qoymaq və /domain/pharmacy/ qovluğu yazmaq kifayətdir. Bütün qalan sistem eyni qalır!

6 Mac terminalında praktika

İndi yuxarıda görüntülədiyimiz strukturu sizin Mac-də yaradacağıq. Bütün əmrləri birər-birər izah edəcəm. Hər kod bloğunun yanındakı 📋 Kopyala düyməsi ilə kodu bufferə alıb Terminal-a yapışdıra bilərsiniz.

6.1. Addım 1 — Terminal-ı açın və masaüstünə keçin

Mac-də Command + Space → "Terminal" yazın → Enter. Sonra masaüstünə keçək:

bash — masaüstünə keçid 
# Masaüstünə keçirik. ~ simvolu — istifadəçi home qovluğudur
cd ~/Desktop

# Təsdiq etmək üçün haradayıq baxaq
pwd

# Nəticə təxminən belə olmalıdır:
# /Users/royatalibova/Desktop
💡
Əmrlər haqqında
  • cd = change directory — qovluq dəyişir
  • ~ = istifadəçinin ev qovluğu (home)
  • pwd = print working directory — haradayıq yazdırır

6.2. Addım 2 — Əsas qovluğu yaradın

bash — əsas qovluğun yaradılması 
# Əsas layihə qovluğunu yaradırıq
mkdir Claude_Modei_Təmir_tikinti

# İçinə keçirik
cd Claude_Modei_Təmir_tikinti

# Yoxlayırıq
pwd
# Nəticə: /Users/royatalibova/Desktop/Claude_Modei_Təmir_tikinti

## 🛠️ Texnologiya Stack-i

- **Backend:** Node.js + Express
- **Frontend:** React + Vite + Tailwind CSS
- **Database:** PostgreSQL 17
- **Dashboard:** R Shiny
- **AI:** Claude API (Anthropic)
- **Process Manager:** PM2
- **VPN:** Tailscale

💡
Commit message konvensiyası

Biz Conventional Commits standartını istifadə edirik:

  • feat: — yeni funksionallıq
  • fix: — xəta düzəlişi
